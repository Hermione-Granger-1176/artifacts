Concepts explained

A token is a chunk of text, somewhere between a character and a word. "unhappiness" might be split into "un", "happiness". "cat" stays as one token. Spaces and punctuation are tokens too.

The model never sees letters. It sees a stream of token IDs, each one a number pointing to an entry in its vocabulary. Every response is the model picking the next token one step at a time.

Rule of thumb: 1 token ≈ 0.75 words in English

Before sampling, the model assigns a raw score to every possible next token. Temperature divides those scores before converting them into probabilities.

Low temperature sharpens the distribution so the top token dominates. High temperature flattens it and gives strange or creative options a better chance.

Code and facts: 0.2 to 0.5. Creative writing: 0.8 to 1.2.

Top P sets a cumulative probability threshold. The model ranks tokens by probability and keeps adding them until the running total reaches P. Everything below the cutoff is excluded.

When the model is confident, the nucleus stays small. When it is uncertain, more tokens survive the cutoff naturally.

0.7 to 0.95 is the usual production sweet spot.

Counting the letters in "strawberry" can fail because a model sees token chunks, not a sequence of individual letters. The letters are present in the text, but they are not always the units the model is processing.

Arithmetic with long numbers is difficult for a related reason. Digits can split into uneven pieces, so the model must reason across chunks rather than line up one digit at a time.

Costs and context windows are measured in tokens too. A compact sentence in English may consume fewer tokens than the same idea written with uncommon text, long numbers, or some non-English scripts.
